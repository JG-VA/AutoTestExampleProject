Main¶

All tests go in here. Also includes data loading and test fixtures

Current tests: test_iteminfo: Checks that names of items and IDs are paired correctly test_searchitems: Checks that search retrieves items test_mainlinks: Checks that links on mainpage work test_carttotal: Checks that cart can add items and calculate total test_cartempty: Checks that cart can be emptied

Paired with prior test to check that cart is emptied successfully

Test to add all Products to cart and check total adds correctly

Search for the first 3 letters of each name and confirm priduct appears in results

class Pages.PageBasic(driver)¶

Parent class with objects on all pages of website

Kewword Arguments:

driver: this one holds the driver for things that need it

infoitems: Items at the bottom of the page, found with ID

Validator¶

class Validator.CommonChecks¶

Holds common methods that return True or False

comparetext: Compares two things

isinlist: Checks if text is among the text elements found by the driver

comparetext(textotcheck,intendedtext)¶

Compares what is in argument1 with argument2

Keyword arguments:

texttocheck: item 1 of comparison

intendedtext: item 2 of comparison

This is made to work with asserts in the main. No type conversions or cleaning,

So make sure you do that before using this. Especially with elements

isinlist(elements,text)¶

Check if text occurs among a series of elements on the page

Keyword arguments:

elements: a list of elements (found by the webdriver) on the page to check the text of

text: Text you are looking for

If the element has no text a placeholder will be added to the list

Returns true if present, false if not

TestData¶

class TestData.LoadCSV¶

Methods for loading test data from a CSV

pricechecker:Converts prices in data to floating point

loadfile: Loads CSV of test data into a dictionary that it returns

loadfile(file)¶

This loads a CSV file of test data, and returns it as a dictionary

Keyword arguments:

file: address of CSV file to load

The CSV file is read as a dictionary by the dictionary reader

It looks for four columns, as follows:

Product Name: Name of item as it appears on website

Price; Price of item on website

Image Path; Unused for now

ID; ID of product in URLs on website

If the file is set up incorrectly, it will print a message instead

pricechecker(inputprice)¶

Attempts to convert price to floating point

Keyword arguments:

inputprice:The price to convert to float

This will try to remove whitespace with regex and remove dollar signs

This was made in case of sloppy test data

It will return a string if it can’t get a float

CommonFunctions¶

class CommonFunctions.MenuText(driver)¶

Function for sending text to an element and click

Keyword arguments:

textfield: element to send the text too

button: element to click after entering text

inputtext: text to be sent to textfield

Initially made for the search bar, could be used for other sequences

class CommonFunctions.Sequences(driver)¶

Superclass to create a chain of actions for the driver

CommonFunctions.find(driver, element)¶

Function for finding an element, usually linked to Pages

Keyword arguments:

driver: pass the webdriver in so it can use it

element: A tuple, the part contains the location method (eg By.ID), second contains

what to be used with it (eg “product-name”)

The element portion was designed to work with the format of the attributes

found in the Pages.py project file

This should handle both single and lists of elements being found

Expect either to come out when writing tests
